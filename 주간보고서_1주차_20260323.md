### 2.4 Docker Compose 인프라 구성 ✅

4개 서비스로 구성된 `docker-compose.yml` 작성 완료:

| 서비스 | 이미지 | 포트 | 역할 |
|--------|--------|------|------|
| `db` | postgres:16-alpine | 5432 (내부) | PostgreSQL 데이터베이스 |
| `backend` | 자체 빌드 | 8080 (내부) | Spring Boot REST API |
| `frontend` | 자체 빌드 | 3000 (내부) | Next.js SSR |
| `nginx` | nginx:alpine | **80 (외부)** | 리버스 프록시 |

- Docker 볼륨: `pgdata` (DB 영속), `uploads` (파일 저장)
- Health check 설정 (`pg_isready`)
- 서비스 간 의존성 정의 (db → backend → frontend → nginx)
- `.env.example` 환경변수 템플릿 작성

### 2.5 Nginx 리버스 프록시 설정 ✅

| 경로 | 라우팅 대상 | 용도 |
|------|-------------|------|
| `/api/` | `backend:8080` | REST API |
| `/api/webhooks/` | `backend:8080` | GitHub/Drive 웹훅 (확장 1) |
| `/uploads/` | `/data/uploads/` (정적) | 파일 다운로드 (30일 캐시) |
| `/` | `frontend:3000` | Next.js SSR |

- `client_max_body_size: 50M` 설정
- 프록시 헤더 전달 (Host, X-Real-IP, X-Forwarded-For/Proto)

### 2.6 Flyway DB 마이그레이션 스크립트 작성 ✅ (V1~V7)

총 **7개 마이그레이션** 스크립트 작성, 전체 MVP 스키마 완성:

| 버전 | 파일명 | 생성 테이블 | 비고 |
|------|--------|-------------|------|
| V1 | `V1__init_users.sql` | `users` | 사용자 (역할: STUDENT/PROFESSOR/TA) |
| V2 | `V2__init_projects.sql` | `projects`, `project_members` | 프로젝트 + 멤버 (동의 필드 포함) |
| V3 | `V3__init_tasks.sql` | `tasks`, `task_assignees` | 칸반 태스크 + 담당자 |
| V4 | `V4__init_meetings.sql` | `meetings`, `meeting_attendees` | 회의록 + 체크인 |
| V5 | `V5__init_activity_logs.sql` | `activity_logs` | 활동 로그 (Score Engine 데이터 소스) |
| V6 | `V6__init_file_vault.sql` | `file_vault`, `tamper_detection_log` | Hash Vault + 불변 트리거 |
| V7 | `V7__init_scores_alerts.sql` | `contribution_scores`, `alerts`, `weight_configs`, `weight_change_log` | 기여도 점수 + 경보 + 가중치 |

주요 설계 포인트:
- **UUID** 기본 키 전략 사용
- `file_vault` 테이블에 **불변 트리거** 적용 (UPDATE/DELETE 차단)
- `activity_logs`에 JSONB `metadata` 필드로 확장성 확보
- `weight_configs`에 **가중치 합계 = 1.00 체크 제약** 적용
- 적절한 인덱스 설계 (검색 빈도 높은 컬럼 대상)

## 3. 미완료 항목 & 차주 계획

### 3.1 이번 주 미완료 (이월)

| 항목 | 사유 | 다음 조치 |
|------|------|-----------|
| `docker compose up -d` 전체 기동 검증 | Docker Desktop 설치 직후, 재부팅 후 PATH 활성화 필요 | 차주 초 Docker 기동 테스트 |
| Flyway 자동 마이그레이션 확인 | Docker 기동 후 검증 예정 | Docker 기동 시 함께 확인 |
| GitHub 리포지토리 생성 | 브랜치 전략 확정 후 진행 (GitHub Flow 선택) | 차주 리포 생성 + 초기 push |
| CI/CD 설정 (GitHub Actions) | 리포지토리 생성 후 진행 | 차주 Docker 빌드 확인 워크플로우 |

### 3.2 차주 목표 (Phase 2 진입)

1. **Docker 전체 스택 기동 검증** — `docker compose up -d` 성공 확인
2. **GitHub 리포지토리 생성** — GitHub Flow 브랜치 전략 적용
3. **백엔드 인증 모듈 구현 시작**
   - User 엔티티 & Repository
   - 회원가입 API (`POST /api/auth/signup`)
   - 로그인 API (`POST /api/auth/login`) — JWT 발급
   - JWT 필터 & SecurityConfig
4. **백엔드 프로젝트 관리 모듈 구현 시작**
   - Project CRUD API
   - 초대 코드 생성 & 참여 API
